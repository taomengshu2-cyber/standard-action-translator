# 为什么不直接把标准扔给普通大模型？

| 比较项 | 普通大模型 | 标准行动转译 Skill |
| --- | --- | --- |
| 标准来源 | 用户临时粘贴，模型临时理解 | 标准被结构化存入标准包 |
| 输出稳定性 | 每次问法不同，结果可能差异较大 | 固定按“标准要求—当前差距—下一步动作”输出 |
| 差距识别 | 依赖模型即时推理，容易泛泛而谈 | 有常见差距规则库 |
| 行动建议 | 容易输出空话 | 必须落到流程、证据、工具、材料、时间或责任人 |
| 边界控制 | 容易被要求代写或评分 | 有防评分、防代写规则 |
| 标准更新 | 每次都要重新解释 | 替换标准包后主流程继续运行 |
| 可迁移性 | 每个学校重新写提示词 | 替换标准包即可迁移 |
| 可验证性 | 难以追踪为什么这样回答 | 可追踪调用了哪条标准、哪条差距规则、哪个行动模板 |

一句话总结：

普通大模型解决的是“这一次能不能答好”；本 Skill 解决的是“每一次、每一条标准、每一个组长都能不能获得稳定、清楚、可执行的反馈”。
